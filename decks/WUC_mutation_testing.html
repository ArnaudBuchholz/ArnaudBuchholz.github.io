Considering the function to test :

function div (a, b) {
  if (b === 0) {
    throw new Error('Do not divide by 0');
  }
  return a / b;
}

And the following test code :

assert.throws(() => div(1,0));
assert.notEqual(div(1,2), 0);

It generates a code coverage of 100%

But what does it really mean ?


All statements are evaluated
during the tests.


It does not guarantee that the code was thoroughly checked.

In particular :

  • The exception is not validated,
    (type and message)
  • The result of the division is not compared with an expected value.

What is mutation testing ?

For each covered statement :

  • The source code is modified with a single mutation to generate a mutant,
  • The tests are executed to check if they still pass.


If a mutant survives the tests,
it means that a defect is not detected !
the tests are not good !

Type of mutations
  • Removing a statement
  • Forcing conditions to true or false
  • Changing operators
  • Changing static content (strings, numbers...)
PROs and CONS of mutation testing
  • Assess the quality of the tests
  • Increase code quality and robustness
  • Resource consuming (time / CPU)
  • Must be limited to unit tests (with proper decoupling to external resources)
