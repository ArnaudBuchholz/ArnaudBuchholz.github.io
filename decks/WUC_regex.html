Regular expressions are only for matching strings
is a common misconception.


It is a powerful too that can also extract values from the matched string.


Regular expressions and capturing groups are tightly integrated in the JavaScript language.

What is a capturing group ?

A pair of parenthesis* define a capturing group. 

const imageDescription = 'Image resolution is 1440×900 pixels.';
const regexpSize = /([0-9]+)x([0-9]+)/;
const match = imageDescription.match(regexpSize);
console.log(`Width: ${match[1]} / Height: ${match[2]}.`);
// expected output: "Width: 1440 / Height: 900."

NOTE : to capture a repeated pattern, make sure the repetition is enclosed inside parenthesis or it will capture only the last occurrence.


* The specific syntax (?: ) defines a non-capturing group.

The result of a match is an array-like where :

  • match[0] contains the matching string,
  • match[1] contains the value of the first capturing group,
  • match[2] contains the value of the second capturing group,
  • ...
  • And additional properties.

[
  '1440x900',
  '1440',
  '900',
  index: 31,
  input: 'This image has a resolution of 1440x900 pixels.'
]

string.replace

Used with string.replace and a replacer function, the capturing group values are provided as additional parameters.

string.replace(regexp, function (match, capturing groups))

string.split

Used with string.split, the resulting array contains the value it was split on. 

"arnaud was here".split(/([aeiouy]+)| /)
[
  '',        'a', 'rn',
  'au',      'd', undefined,
  'w',       'a', 's',
  undefined, 'h', 'e',
  'r',       'e', ''
]
