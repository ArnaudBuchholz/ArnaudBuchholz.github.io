Things to consider when moving to GitHub

So you have been writing a nice and helpful piece of library and you want to move it to GitHub to expose it to the world. Here is a list of things to consider before jumping into this wild adventure.
## Setting the expectations First of all, let's talk about the obvious benefits of using [GitHub](https://github.com/): * your code is in a safe location * it is shared immediately: everyone can see your changes * you have branching & merging features * the website offers interesting statistics on your project * you have an inbuilt However, the code is public by default.
