Actually, I might be wrong and this is OK, we all [learn from our mistakes](http://scottberkun.com/essays/44-how-to-learn-from-your-mistakes/). * Obviously, this has nothing to do with my professional activity: I studied this project on my personal time. ## The project Today I will focus on [PowerCSS](https://github.com/mmikowski/powercss). Long story short, the author proposes to replace static CSS files with dynamic information maintained by JavaScript. The advantages are multiple: full control on CSS properties, dynamism and compression. ## About the idea I like the idea of being able to generate CSS using JavaScript. The README.md is quite complete, I would even say a little bit too exhaustive: it looks like doing simple things is extremely complex. I would probably simplify it to be more introductive and create a separate documentation page for 'advance' use. I regret that the samples do not really illustrate the reason why JS should be preferred over static CSS. As far as I know, you can easily switch CSS inside a web page. ## Coding style _xxx_ everywhere !
